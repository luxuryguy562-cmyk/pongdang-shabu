# 🏢 퐁당샤브 AI 에이전트 팀 — 사용 가이드

## 사장님이 하는 일 (딱 3가지)

**1️⃣ 요청하기** — "월별 매출 비교 차트 추가해줘"
**2️⃣ 계획서 승인** — "OK" / "이것도 추가해" / "이건 안 해도 돼"
**3️⃣ 결과 확인** — pongdang-shabu.pages.dev 접속

## 에이전트 팀 소개

| 에이전트 | 성격 | 하는 일 |
|---------|------|---------|
| context_reader | 꼼꼼한 탐정 | 현재 코드 상태 파악 |
| planner | 똑똑한 PM | 계획서 작성 |
| designer | 까다로운 디자이너 | 화면 설계 (쓰기 편하게) |
| reviewer | 의심 많은 QA | 빠진 것, 위험한 것 체크 |
| coder | 묵묵한 개발자 | 코드 구현 |
| tester | 강박적 검증자 | 에러 확인 |
| deployer | 신중한 배포자 | push + 사이트 확인 |

## 작업 규모에 따라 경로가 다릅니다

**간단한 거** (오타 수정, 색상 변경): 계획 한 줄 → 바로 수정 → 배포
**보통 거** (기능 개선, UI 추가): 전체 팀 가동 → 계획서 → 승인 → 개발 → 배포
**큰 거** (새 모듈, DB 변경): 전체 팀 + 상세 계획서 + DB 변경 SQL까지 보여드림 → 승인 → 개발 → 배포
