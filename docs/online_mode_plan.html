🔍 빙산 발견 (헌법 1-8)

2026-05-26에 거래처 → 쿠팡 경로로 쿠팡 처리 기능이 이미 만들어져 있습니다. 사장님이 이번에 영수증 탭에서 쿠팡을 분석하셔서, 그 기능을 안 거치고 일반 영수증으로 읽혀 결과가 엉망이 된 겁니다.

이미 있는 쿠팡 기능 (거래처 → 쿠팡 진입 시):

  • 📸 쿠팡 스샷 → AI 분석 — 전용 규칙으로 주문 추출 (sidemenu.js)
  • 📥 분류 대기함 — AI가 추천한 카테고리를 사장님이 확인 후 저장 (coupang_inbox 표)
  • 🔖 북마클릿 자동 동기화 — 쿠팡 사이트에서 주문을 자동으로 가져옴 (Edge Function)

→ 저장 위치도 다릅니다. 기존 쿠팡 = 주문 내역(vendor_orders) / 영수증 탭 = 영수증(receipts).

✅ 핵심 효과 — 이름이 짝이 맞습니다

온라인 모드는 상호를 "쿠팡"으로 고정하고, 실제 판매자(예: 철물인)는 품목명에 남깁니다. 그러면 카드·통장 내역의 "쿠팡"과 영수증의 "쿠팡"이 같은 이름으로 묶여서 한 화면에 보입니다. (직구/온라인 화면은 상호 글자로 묶는 구조라 가능 — 코드 확인 완료)

❓ 사장님 결정 필요 — 기존 쿠팡 기능을 어떻게 할까요

온라인 모드를 새로 만들면 거래처 → 쿠팡 스샷 분석과 겹칩니다. 잔재로 남으면 헷갈리니(헌법 1-6) 정리 방향을 정해야 합니다.

방법 A — 영수증 탭 온라인 모드로 일원화 CTO 추천
  • 온라인 구매는 전부 영수증 탭 → 온라인에서 처리 (사장님이 자연스럽게 들어간 그 자리)
  • 기존 거래처 → 쿠팡 스샷 분석 버튼은 제거 (중복 정리)
  • 북마클릿 자동 동기화는 유지 (사장님 손 안 가는 자동화라 가치 큼 — 비전)
  • ⚠️ 저장 위치 통일 필요: 영수증(receipts)로 모을지, 주문내역(vendor_orders)로 모을지 — 구현 시 한쪽으로
방법 B — 둘 다 유지
  • 거래처 → 쿠팡(자동 동기화·대기함) + 영수증 탭 온라인(스샷 즉석) 둘 다 둠
  • ⚠️ 같은 쿠팡 주문을 두 군데서 입력 가능 → 중복 위험 + 사장님 헷갈림

✅ 표 구조 변경 없음

온라인 영수증은 기존 영수증 표(receipts)의 칸을 그대로 씁니다 — 상호(vendor)에 "쿠팡", 분류(category_id)는 품목별로. 새 칸·새 표 안 만듭니다. (방법 A에서 저장 위치를 vendor_orders로 정하면 그건 기존 표라 역시 변경 없음)

🛡️ 기존 기능 안전 (회귀 방지)

  • 마트·시장 = 이름만 바뀌고 동작은 100% 동일 (기존 직구 영수증 그대로 보임)
  • 거래처 모드 = 손대지 않음
  • 이미 저장된 영수증 = 영향 없음

⚡ 리스크

  • "직구" 글자가 코드 여러 곳에 흩어져 있음 → 한 군데라도 놓치면 옛 이름 잔재. 전부 찾아 바꾸고 검증(헌법 11조)
  • 방법 A에서 기존 쿠팡 스샷 기능 제거 시 → 그 흐름에 익숙한 동작 끊김. 자동 동기화는 살려서 완충
  • 온라인 플랫폼 목록 = 코드에 박으면 나중에 추가 번거로움 → "기타" 직접 입력으로 보완
