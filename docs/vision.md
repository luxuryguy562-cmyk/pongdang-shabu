## 한 줄 정의

> **식당 사장님이 정산에 손 하나 까딱 안 해도 매출·지출·순익이 직관적으로 보이는 외식업 통합 자동화 플랫폼.**

---

## 1. 이 앱이 푸는 진짜 문제

식당 사장님이 매월 손으로 한 일:
- 계좌·카드 내역 인터넷에서 엑셀 받아서 복붙
- 영수증(거래처·마트) 일일이 적기
- 모든 항목 분류 수동
- 다 했는데도 **얼마 남았는지 한눈에 안 보임**

**= 매일 동기화·정산 = 사장님 월 20시간 노가다.**
AI 있는 시대에 사장님이 이걸 손으로 하는 건 말이 안 됨.

---

## 2. 시중 앱 한계 + 우리 차별점

### 시중 앱 한계
- 입력 자동화 일부 됨
- BUT **"내가 보고 싶은 게 안 나옴"** → 매출/지출/남음 한 화면 직관 X

### 🎯 우리 차별점 (한 줄)
> **"캐시노트는 매출 SaaS, 우리는 순익 SaaS"**
> 캐시노트는 "얼마 벌었나" 답함. 우리는 **"얼마 남았나"** 답함.

### 우리 무기 1~3순위
| 순위 | 무기 |
|---|---|
| 🥇 | **순익 한 화면** (매출-지출-남음 직관) |
| 🥈 | **지출 자동화 압도** (영수증·거래처 명세 OCR — 캐시노트 약점) |
| 🥉 | **양면 시장 5개 + 커뮤니티** (캐시노트는 대출 1개) |

---

## 3. 1순위 가치 — 시간 + 측정 지표

### 헌장 1조 (사장님 명언)
> *"나는 정산에 있어서 손 하나 까딱하기 싫다. 그게 최고의 목적."*

### 측정 지표 (영업 무기)
> **사장님 본인 정산 시간: 월 20시간 → 월 30분 (97.5% 절감)**
> 모든 영업·마케팅 자료에 박는 숫자.

**→ 모든 작업·기능·결정은 이 한 줄로 판단. 사장님 손이 가야 하는 기능 = 실패.**

---

## 4. 타겟·로드맵

### 타겟
- **Phase 1**: 식당 (퐁당샤브 = PoC)
- **Phase 2+**: 외식업 전체 (카페·술집·치킨·배달·푸드트럭)

### 로드맵 (Phase별 동시 진행)
| Phase | 무엇 | 시점 |
|---|---|---|
| **1. PoC** | 정산 + 거래처 명세 자동화 완벽 | 2개월 내 |
| **2. 사용자 모집** | B2C 마케팅 + B2B 영업 + 베타 10명 시장 검증 | Phase 1 동시 |
| **3. 플랫폼화** | 인접 5개 도메인 슬롯 + 단계적 입점 | Phase 2 동시 |
| **4. 커뮤니티** | 사장님 정보 공유방 (토스 주식 모델) | Phase 3 동시 |

### 인접 5개 도메인 (Phase 3) + 양면 시장
| 도메인 | API 연계 시 | 마케팅 |
|---|---|---|
| 🥬 식자재 | 발주·명세 자동 | "이 업체 쓰면 명세 자동" |
| 📑 세무 | 자동 신고 | "이 세무사 쓰면 신고 끝" |
| 👷 노무 | 4대보험 자동 | "이 노무사 쓰면 보험 자동" |
| 🛡️ 보험 | 매출 기반 견적 | "이 보험 쓰면 30% 할인" |
| 💰 대출 | 실시간 한도 | "이 캐피탈 쓰면 5분 한도" |

→ **선순환**: 입점 ↑ → 자동화 ↑ → 사용자 ↑ → 입점 ↑

### PoC 완벽 기준
**우리 가게 1개월 스프레드 수동값 = 앱 자동값 100% 일치** (매출·지출·분류 3개 모두)

### 목표
- **1년**: 사용자 1,000명 / 매출 5천만원 (Phase 1+2)
- **2~3년**: 수수료 본격화 (Phase 3+)

### 수익 모델 (다층)
1. SaaS 구독료
2. 입점 업체 수수료 (식자재·세무·노무·보험)
3. **대출 leads 수수료** ← 캐시노트 1,000억 모델
4. 광고 (Phase 4 이후)

### 🛠️ 기술 스택 — 자동화 진짜 답 (2026-05-24 사장님 명시 큰 방향)

> 옛 가정 (Codef API) 폐기. 사장님 직관 = **문자 자동 읽기 + 매출 백그라운드**.

| 자동화 영역 | 우선 | 기술 |
|---|---|---|
| 카드·계좌 입출금 | 🥇 | 🤖 **안드로이드 SMS 자동 읽기** (Capacitor 래핑, 권한 1회 = 영구 자동) |
| iOS 보완 / 카드사 명세 | 🥈 | 📧 **Gmail OAuth 백그라운드** (영수증·명세서 이메일 자동 인식) |
| 매출 백그라운드 | 🥉 | 💳 **POS API** (`store_settings.ups_*` 업솔루션 연동 강화) |
| iOS SMS | ⏸️ | Phase 2+ (Apple 정책 불가, 이메일·POS만) |

→ 캐시노트도 이 길로 갔음. 한국 식당 사장 70%+ 안드 = 본진. 단일 SPA(헌법 1-4) 유지 + Capacitor로 네이티브 권한만 추가.

## 5. 루트 원칙 — 라벨링 + 능동 제안

### 5-1. 거절 아니라 **라벨링**
- 🟢 **루트 안** — 식당 사장님 일상 (정산·식자재·세무·노무·보험·대출·커뮤니티)
- 🟡 **루트 옆** — 식당 외 사장님 개인에 도움
- 🔴 **루트 밖** — 식당 무관
- 🟢 강추 / 🟡 사장님 판단 / 🔴 Claude 먼저 말림

### 5-2. 능동 제안 의무 (Claude 책임)
사장님 모르는 기능·세무·기술 트렌드 = **Claude가 챙겨 먼저 제안 의무**. 시킨 것만 = 직무유기.

### 5-3. 분업
| 영역 | 누가 압도? |
|---|---|
| 식당 운영·매출·고객·현장 | 사장님 |
| 세무·법무·기술·DB·UX·기능 트렌드 | Claude |
| 비전·방향·최종 결정 | 사장님 |

## 6. 운영 효율성 — 최소 직원 원칙

### 6-1. 동일 철학
사장님 1조 = **B2C 가치 + 회사 운영 둘 다 적용**. 운영 직원 최소화 = 수익성.

### 6-2. 직원 부르기 전 4단계 (자동 해결 우선)
1. **자가 치유** — AI 먼저 (직원 모름)
2. **사용자 셀프** — AI 챗봇 + FAQ
3. **Slack 신호등 푸시** — 직원 폰으로 처리
4. **운영 대시보드** — 안 풀린 것만

### 6-3. 운영 대시보드 항목 (신호등)
🔴 오류 / 💬 QnA / 💰 AI 비용 / 🗄️ DB 비용 / 📊 매출·구독자 / ⚙️ 배포 상태

### 6-4. KPI
Phase 2 (1,000명) 시 운영 직원 **1~2명**으로 굴러가야. 그 이상 = 6-2 자동화 실패 = 1조 위반.

## 7. 약속·보장 (신뢰 + retention 무기)

### 7-1. 🔒 데이터 신뢰·정책
- 사장님 데이터 **명시 동의 없이 외부 제공 X**
- **AI 학습에 사용자 데이터 안 쓴다** (외부 AI 옵트아웃)
- **해지·폐업 시 30일 보존 후 완전 삭제** (다운로드 권리)
- Phase 3 leads 판매 = **익명화 통계** 또는 **건별 동의**

### 7-2. ⏱️ 신뢰성·가용성 (SLA)
- 가용성 **99.5%** (월 다운 3시간 이하)
- **마감 시간대 (밤 10시~새벽 2시) 다운 절대 금지**
- Sentry 실시간 감지 (6번 연동)

### 7-3. 🚀 자동 온보딩
- **신규 가입 5분 안 첫 영수증 자동 인식 성공** 보장
- 단계별 가이드 (정산 → 분석 → 인사이트)
- 막히면 AI 챗봇 (6-2의 2번 연결)

### 7-4. 🧠 자동 인사이트 (AI 직원)
- 사장님이 안 봐도 어플이 알려줌:
  - "이번 달 인건비 35% — 업계 평균 +5%"
  - "식자재 ○○ 단가 5% 인상 — ○○가 8% 저렴 (Phase 3 입점)"
- = **AI 직원 1명 24시간 무료 가동**

## 📌 모든 agent에게 (자문 의무 5개)

매 작업 시작 전 자동 자문:

1. **사장님 손이 가는가?** → 가면 자동화 더 가능한가
2. **사장님이 직관적으로 보는가?** → 헷갈리면 designer 재호출
3. **다른 외식업 사장님에게 팔 수 있는가?** → 우리 가게 하드코딩 = 빨간불
4. **사용자 신뢰 깨는가?** → 데이터·SLA 자동 점검
5. **루트에서 벗어났는가?** → 5-1 라벨링 적용
