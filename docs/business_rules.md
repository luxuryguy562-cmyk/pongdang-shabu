## 1. 인건비는 익월 지급
**날짜**: 2026-04-09  
**규칙**: 3월 근무 급여는 4월에 출금됨  
**영향**:
- mydata_transactions에서 "급여03월탁성현" → `attribution_month = 2026-03`
- 대시보드 3월 조회 시 → 4월 출금이지만 3월 비용으로 집계
- 가마감(3월): attendance_logs 3월 계산 급여
- 진마감(3월): mydata_transactions에서 attribution_month='2026-03'인 건

## 2. 가마감 vs 진마감
**날짜**: 2026-04-09  
**규칙**:
- **가마감** = 발생기준(예측). 아직 출금 안 됐어도 발생한 비용 포함
  - 인건비: 근태 기록 기반 예상 급여
  - 로열티: 매출 × 요율
  - 공과금: 예상 금액
- **진마감** = 출금기준(확정). 실제 통장에서 빠진 금액만
  - 인건비: 실제 지급된 통장 출금액
  - 로열티: 실제 출금액 (유림에퐁당)
  - 공과금: 실제 출금액

## 3. 은행 엑셀 구조 (신한기업뱅킹 기준)
**날짜**: 2026-04-09  
**규칙**:
- "적요" 컬럼 = 거래 유형 코드 (타행PC, FB자금, BZ뱅크 등) → 분류에 쓸모없음
- "내용" 컬럼 = 실제 거래처/내용 (NH17702198, 급여03월탁성현 등) → 분류에 사용
- 분류 시 **"내용" 컬럼 우선**, "적요"는 보조
- 합계 행이 마지막에 있음 → 자동 스킵 필요

## 4. 정산 제외 항목
**날짜**: 2026-04-09  
**규칙**: 다음 항목은 지출 집계에서 제외 (exclude_from_settlement = true)
- 카드대금 (롯데카드(주), 신한카드) — 카드사에 갚는 돈은 이미 카드결제 시 비용 처리됨
- 배당금 (탁병근, 김은성, 탁성현) — 비용이 아닌 이익 배분
- 개인결제 (이송은 소액) — 개인 상계 건

## 7. 권한 체계 (auth_level)
**날짜**: 2026-04-13
**규칙**: employees.auth_level 4단계

| auth_level | 누구 | 범위 | 자동 로그인 | PIN |
|---|---|---|---|---|
| owner | 앱 운영자 (사장님) | 전 프랜차이즈, 전 매장 | O | 필수 |
| franchise_admin | 브랜드 관리자 | 해당 프랜차이즈 전 매장 | X | 필수 |
| store_manager | 매장 관리자 (점장/팀장) | 해당 매장 관리 기능 | X | 필수 |
| staff | 매장 직원 | 해당 매장 제한 기능만 | X | 필수 |

**UI 접근 규칙**:
- staff: 영수증, 근태, 근무계획, 마감정산만
- store_manager 이상: + 대시보드, 지출리스트, 햄버거 메뉴 (거래처/고정비/설정/직원관리)
- owner: 전체 + 매장 전환

**호환**: 기존 is_manager=true → auth_level='store_manager'로 자동 매핑
