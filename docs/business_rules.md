## 1. 인건비는 익월 지급
**날짜**: 2026-04-09  
**규칙**: 3월 근무 급여는 4월에 출금됨  
**영향**:
- mydata_transactions에서 "급여03월탁성현" → `attribution_month = 2026-03`

- 가마감(3월): attendance_logs 3월 계산 급여
- 진마감(3월): mydata_transactions에서 attribution_month='2026-03'인 건

## 2. 가마감 vs 진마감
**날짜**: 2026-04-09  
**규칙**:
- **가마감** = 발생기준(예측). 아직 출금 안 됐어도 발생한 비용 포함
  - 인건비: 근태 기록 기반 예상 급여
  - 로열티: 매출 × 요율
  - 공과금: 예상 금액
- **진마감** = 출금기준(확정). 실제 통장에서 빠진 금액만
  - 인건비: 실제 지급된 통장 출금액
  - 로열티: 실제 출금액 (유림에퐁당)
  - 공과금: 실제 출금액

## 3. 은행 엑셀 구조 (신한기업뱅킹 기준)
**날짜**: 2026-04-09  
**규칙**:
- "적요" 컬럼 = 거래 유형 코드 (타행PC, FB자금, BZ뱅크 등) → 분류에 쓸모없음
- "내용" 컬럼 = 실제 거래처/내용 (NH17702198, 급여03월탁성현 등) → 분류에 사용
- 분류 시 **"내용" 컬럼 우선**, "적요"는 보조
- 합계 행이 마지막에 있음 → 자동 스킵 필요

## 4. 정산 제외 항목
**날짜**: 2026-04-09  
**규칙**: 다음 항목은 지출 집계에서 제외 (exclude_from_settlement = true)

## 7. 권한 체계 (auth_level)
**날짜**: 2026-04-13 / **호칭 정정**: 2026-05-05 / **호칭 재확정**: 2026-05-05 (super_admin 폐기)
**규칙**: employees.auth_level 4단계

| auth_level | 호칭 (UI 노출용) | 누구 | 범위 | 자동 로그인 | PIN |
|---|---|---|---|---|---|
| owner | **👑 사장** | 매장 운영 사장 (각 매장 1명) | 해당 매장 전체 | O | 필수 |
| franchise_admin | 본사 관리자 | 본사 관리자 | 해당 프랜차이즈 전 매장 | O | 필수 |
| store_manager | 매장 관리자 (점장/팀장) | 점장/팀장 | 해당 매장 관리 기능 | O | 필수 |
| staff | 일반 직원 (시급제) | 직원 | 해당 매장 제한 기능만 | O | 필수 |

**자동 로그인 규칙 (2026-05-06 갱신)**:
- 모든 권한이 동일하게 자동 로그인 가능 (본인 폰 가정)
- 한 번 PIN으로 로그인하면 localStorage(`pd_emp`)에 저장 → 다음 진입 시 자동
- 로그아웃하면 localStorage 클리어 → 다시 PIN 입력 필요
- 직원 fingerprint(기기 인증)는 출퇴근 시점에 별도 검증 (dev_lessons #54)

### ⚠️ 호칭 절대 규칙 (2026-05-05 재확정)
- **owner = 매장 운영 사장** (이송은 같은 사람). UI 라벨은 "👑 사장".
- **앱 개발자/운영자(김은성=사용자)는 employees 테이블에 없음.** 매장 데이터와 완전 분리.
  - 시스템 수정은 코드 변경 + 배포 또는 Supabase 콘솔로 직접 SQL.
  - 매장 UI를 통한 super_admin 로그인 메커니즘은 **불필요 → 폐기 결정** (2026-05-05).
- 가입 화면 owner 입력 라벨: **"사장 이름"**. ("사장님" 존칭 금지, "총관리자" 표기 금지)
- 같은 실수 반복 시 `dev_lessons.md`에 항목 추가하고 헌법에 못박을 것.

### 이력
- 2026-04-13: auth_level 4단계 도입. owner=앱 운영자 라벨로 시작.
- 2026-05-05 1차 정정: 사용자(김은성=앱 개발자)를 employees.owner로 박은 게 잘못 → "총관리자" 라벨로 변경 (사장 호칭 금지).
- 2026-05-05 2차 정정: 앱 개발자가 매장 직원 테이블에 있는 구조 자체가 오설계 → 김은성 row 삭제, 이송은을 owner 승격, 라벨 "👑 사장"으로 환원. owner = 매장 운영 사장.

**UI 접근 규칙**:
- staff: 영수증, 근태, 근무계획, 마감정산만
- store_manager 이상: + 대시보드, 지출리스트, 햄버거 메뉴 (거래처/고정비/설정/직원관리)
- owner: 전체 + 매장 전환

**호환**: 기존 is_manager=true → auth_level='store_manager'로 자동 매핑
