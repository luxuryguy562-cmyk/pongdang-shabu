> 코드 변경 시 이 규칙에 위배되는지 확인할 것.

## 1. 인건비는 익월 지급
**날짜**: 2026-04-09  
**규칙**: 3월 근무 급여는 4월에 출금됨  
**영향**:
- mydata_transactions에서 "급여03월탁성현" → `attribution_month = 2026-03`
- 대시보드 3월 조회 시 → 4월 출금이지만 3월 비용으로 집계
- 가마감(3월): attendance_logs 3월 계산 급여
- 진마감(3월): mydata_transactions에서 attribution_month='2026-03'인 건

## 2. 가마감 vs 진마감
**날짜**: 2026-04-09  
**규칙**:
- **가마감** = 발생기준(예측). 아직 출금 안 됐어도 발생한 비용 포함
  - 인건비: 근태 기록 기반 예상 급여
  - 로열티: 매출 × 요율
  - 공과금: 예상 금액
- **진마감** = 출금기준(확정). 실제 통장에서 빠진 금액만
  - 인건비: 실제 지급된 통장 출금액
  - 로열티: 실제 출금액 (유림에퐁당)
  - 공과금: 실제 출금액

## 3. 은행 엑셀 구조 (신한기업뱅킹 기준)
**날짜**: 2026-04-09  
**규칙**:
- "적요" 컬럼 = 거래 유형 코드 (타행PC, FB자금, BZ뱅크 등) → 분류에 쓸모없음
- "내용" 컬럼 = 실제 거래처/내용 (NH17702198, 급여03월탁성현 등) → 분류에 사용
- 분류 시 **"내용" 컬럼 우선**, "적요"는 보조
- 합계 행이 마지막에 있음 → 자동 스킵 필요

## 4. 정산 제외 항목
**날짜**: 2026-04-09  
**규칙**: 다음 항목은 지출 집계에서 제외 (exclude_from_settlement = true)
- 카드대금 (롯데카드(주), 신한카드) — 카드사에 갚는 돈은 이미 카드결제 시 비용 처리됨
- 배당금 (탁병근, 김은성, 탁성현) — 비용이 아닌 이익 배분
- 개인결제 (이송은 소액) — 개인 상계 건
