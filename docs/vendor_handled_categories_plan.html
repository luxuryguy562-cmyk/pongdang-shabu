🟢 루트 안 — 검수 부담↓·정확도↑, 비전 일치

📌 한 줄 요약

거래처마다 취급품목(여러 개 가능)을 정해두면, 그 거래처 영수증은 정한 품목 안에서만 분류됩니다. 온라인·마트는 지금처럼 AI 자율. 사장님이 일일이 안 봐도 되고, 성원치킨너겟 같은 오분류가 사라집니다.

🎯 동작 (사장님 확정안)

종류카테고리 방식
🏪 거래처취급품목 정함(편집 가능) → 그 안에서만 분류. 1개면 고정(AI 분류 생략)
🌐 온라인자율 — AI가 전체에서 품목별
🛒 마트·시장자율 — AI가 전체에서 품목별
✅ 효과
  • 정확도 — 성원치킨너겟을 육류로 틀릴 일 없음 (취급품목이 공산품이면 그 안에서만)
  • 검수 부담↓ — 2~3개 안에서만 정해지니 안심
  • 단일 거래처(정육점=육류) — AI 분류 아예 생략, 고정 (비용·부하↓)

🖥️ 거래처 편집 화면 변경

지금은 대분류·소분류 1개만 고르는 칸입니다. 이걸 취급품목 여러 개 체크로 바꿉니다.

거래처 편집 — 웰스토리
취급품목 (사장님이 사는 것만 · 카테고리는 DB에서 자동 표시):
📦 식자재 대분류 전체
✓ 육류 ✓ 공산품 야채 수산
🧹 비품 대분류 전체
→ 웰스토리 영수증은 육류·공산품 중에서만 분류됩니다
※ 세금·고정비·공과금은 "구매"가 아니라 후보에 안 나옴. 카테고리 이름·목록은 사장님 매장 DB 그대로(하드코딩 X)
선택 방식 (사장님 지적 반영)
  • 대분류 전체 — "식자재 전체" 누르면 그 아래 육류·야채·공산품·수산 다 취급
  • 소분류만 — "식자재 > 육류·공산품"처럼 일부만 (야채 빼고)
  • 후보 목록 = DB에서 동적 — 사장님이 카테고리 추가/이름변경하면 자동 반영
  • 지출(구매) 카테고리만 — 세금·고정비·공과금·인건비 자동 제외

🗄️ DB(데이터 표) 변경

거래처 표에 "취급품목" 칸 1개 추가

거래처(vendors) 표에 handled_category_ids(취급품목 목록) 칸 신설. 카테고리 여러 개를 목록으로 저장.

✅ 기존 거래처 자동 이전 — 안 깨짐

지금 거래처들은 분류가 1개씩 박혀 있습니다(프레시원=식자재 등). 이걸 취급품목 1개로 자동 복사합니다. → 기존 거래처는 그대로 "그 분류로 고정" 상태가 되어 동작이 안 바뀝니다. 사장님이 나중에 품목을 추가하고 싶으면 편집.

실행 SQL: ALTER TABLE vendors ADD COLUMN handled_category_ids JSONB; + 기존 category_id를 목록으로 복사
롤백 SQL: ALTER TABLE vendors DROP COLUMN handled_category_ids;
→ 사장님 "실행 승인" 4글자 받고 실행 (헌법 8조)
