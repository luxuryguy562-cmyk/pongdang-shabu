## 🎯 메인 페르소나 — Phase 1 (PoC + 베타 모집)

### 김사장님 (가칭) — **자영업 식당 사장 1인 운영형**

| 항목 | 값 |
|---|---|
| 나이대 | 35~55세 (⚠️ 사장님 검증 — 본인 직관 vs 평균) |
| 성별 | 무관 |
| 직책 | 식당 대표 (직접 운영 + 직접 정산) |
| 매장 규모 | 직원 1~10명 / 월 매출 1,000만~1억원 |
| 디지털 친숙도 | 스프레드시트 쓸 수 있는 수준 (엑셀·구글시트 직접 조작) |
| 위치 | 한국 전국 (지방 도시 포함) |

### 페인포인트 (vision 1번 기반)
1. **매일 동기화 노가다** — 계좌·카드 내역 매일 인터넷에서 받아 복붙
2. **분류 노가다** — 거래·영수증 항목 일일이 분류
3. **순익 안 보임** — 한 달 끝나면 "얼마 남았나" 직관 X
4. **매월 20시간 정산 시간 도둑**

### 가치 추구 (vision 3·5번 기반)
- **시간 자동화** (1순위)
- **직관적 화면** (보자마자 이거구나)
- **데이터 신뢰** (내 영업비밀 새지 않음)
- **저비용** (캐시노트보다 싸거나 동급)

### 디바이스 (2026-05-24 사장님 답)
- **모바일 + PC 둘 다 사용** (사장님 명시)
- 모바일: 식당 현장에서 폰으로 확인 (영수증 사진·출퇴근·실시간)
- PC: 마감 시 PC로 자세히 (월 정산·엑셀·노무 출력)
- → UI는 **모바일 우선**, PC는 큰 화면 활용 (사이드 패널 등 가능)

### 사용 시간대
- **밤 10시~새벽 2시** = 마감·정산 골든타임 (vision 7-2)
- 낮 시간 = 영수증 사진 찍기 (틈틈이)

### 결제 의지 (⚠️ 사장님 검증 필요)
- 월 1만원 = 무조건 결제
- 월 3만원 = 캐시노트 대비 가치 있으면 결제
- 월 5만원 = 양면 시장 보너스(자동화) 있을 때만
- → `pricing.md`에서 다시 정함

## 📊 서브 페르소나 — Phase 2+ (외식업 확장)

### 1. 박매니저 (가칭) — **프랜차이즈 매니저형**
- 본사 직속 매니저 / 매장당 1명 / 정산만 담당
- 디지털 친숙도 ↑
- 결제 = 본사 일괄 (B2B)

### 2. 이바리스타 (가칭) — **카페 사장형**
- 1인 사장 또는 2~3인 매장
- 영수증 적음, 매출 카드 위주
- 디지털 친숙도 ↑

### 3. 최치킨 (가칭) — **배달 전문형**
- 배달 앱 매출 위주
- 영수증 적음, 결산은 배달앱 정산 위주
- 외주 세무사 의존도 ↑

### 4. 정포차 (가칭) — **술집·포차형**
- 늦은 시간 마감
- 현금 비중 ↑ (vision 7-2 마감시간 SLA 가장 중요)
- 직원 회전율 ↑

### 5. 송푸드트럭 (가칭) — **푸드트럭/이동형**
- 위치 자주 변동
- 모바일 100%
- 매출 적지만 횟수 ↑

## 🚫 우리 사용자 아닌 사람 (안티-페르소나)

| 타입 | 왜 안 맞나 |
|---|---|
| 대형 프랜차이즈 본사 | 자체 ERP 사용, B2B 영업 제한적 |
| 호텔·뷔페·웨딩홀 | 정산 구조 다름 (객실·룸·이벤트) |
| 무인 매장 | 사람 손 자체가 적음 (페인포인트 X) |
| 일반 소매업 (옷·전자제품) | 외식업 도메인 안 맞음 |
| 직접 정산 안 하는 사장님 | 우리 가치(시간 절감) 와닿지 않음 |

## ⚠️ 사장님 검증 필요 (3가지)

매번 답하기 부담스러우면 한꺼번에 짧게라도 OK:
