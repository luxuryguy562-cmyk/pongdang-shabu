## 🔜 [최우선 — 내 매장들 "전체 순익" 구현 (회계 핵심)]

> 멀티매장 통합은 매출까지 완료(서버창구 my-stores + 전환 switch-store + 화면 + 로그인 후 자동진입). **순익만 남음.** 상세 설계는 `blueprint.md` "순익 — 미구현" 섹션 참조.

- **요구**: "내 매장들"에서 매장별 **순익이 항상** 보여야 함(저장 방식 X — 사장님 "들어가야만 보이면 별로"). → 서버 계산.
- **왜 안 끝냈나(정직)**: 순익 = 매출 − (식자재+인건비+고정비+로열티+카드수수료). **인건비가 주휴수당 포함 0-7 단일함수**라 서버 재현이 회계 로직 절반 복제 = 거대 + 7-7 위험. 어설프게 박으면 대시보드 순익과 어긋나 회계 깨짐.
- **할 일(순서)**: ① 대시보드 인건비/순익 계산 정확히 파악 → ② my-stores에 비용 5종 집계 추가(인건비 주휴 포함) → ③ **검증 안전핀**: 논산점 "대시보드 순익 == 내 매장들 순익" 일치 확인, 어긋나면 막음 → ④ 프론트 순익 표시("—" 자리 채움).
- **테스트 매장**: 천안점(`store_id 9a2094a9-90e3-4bf8-965d-eaca6fa55392`, code TEST2HO)을 **검증용으로 남겨둠**(사장님 "지속 체킹"). 순익 검증 끝나면 정리.
